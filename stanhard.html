    前書き

    現段階のプログラミング入門演習の授業で扱ったことだけで解ける問題を作成しました。

    • 先ず何をすればいいかわからない。
    • どう解けばいいかわからない。
    • どう書けばいいかわからない。
    を解決する自分なりの方法を早い内に発見し、胸を張ってプログラムを書くことが出来ると言えるようになりましょう。


    どちらの問題も配列を使わなくても解くことが出来ます。が、私も上級の問題を配列ナシはめんどくさくてやりたくないです。 中級は配列を使っても使わなくても簡単に解くことが出来ます。是非、どちらもやってみてください。


    解けなくても投げ出さないでください。
    きっと出来るようになりますから。


    また、解説及びサンプルコード（C,Python）を公開します。


  • 中級

    1 の位をグループに分けて考えると 1 の位が、
    『１』は 1 と 11、『２』は 2 と 12、『３』は 3 と 13、『４』は 4 と 14、『５』は 5 と 15、『６』は 6、『７』は 7、『８』は 8、『９』は 9、『０』は 10、のようになります。
    また、数字の 1 の位は 10 で割った余りで得ることが出来ます。


  • 上級

    • n と n 次正方行列はキーボードから受け取った数字とします。(標準入力)
    掃き出し法 (行基本変形) を行い階段行列を出力してください。
    また、解が一意に定まらない場合は”undefined”と出力してください。

    入力形式 n
    a11　a12　a13　…　a1n　b1
    a21　a22　a23　…　a2n　b2
    a31　a32　a33　…　a3n　b3
    ︙
    an1　an2　an3　…　ann　bn
    出力形式 c11　c12　c13　…　c1n　d1
    c21　c22　c23　…　c2n　d2
    c31　c32　c33　…　c3n　d3
    ︙
    cn1　cn2　cn3　…　cnn　dn

    入力例１ 3
    1 -2 -3 4
    2 3 4 4
    3 -4 -7 10
    出力例１ 1 0 0 3.00000
    0 1 0 -2.00000
    0 0 1 1.00000

    入力例１は、以下の数式と同じ入出力であることを確認してください。
    また、数字と数字の間は半角スペースです。

    1 -2 -3 4 2 3 4 4 3 -4 -7 10 → 1 0 0 3 0 1 0 -2 0 0 1 1

    ベクトルと行列の授業を取ってなかったらごめんなさい。
    ネット上に掃き出し法(行基本変形)についての解説記事はたくさんあるのでそれらを参考にしてください。


    入力例２ 4
    1 -1 1 -1 4
    1 1 1 1 2
    -1 -1 -1 0 3
    -1 1 1 1 0
    出力例２ 1 0 0 0 1.00000
    0 1 0 0 -6.00000
    0 0 1 0 2.00000
    0 0 0 1 5.00000

    階段行列を導くために行列の入れ替えが発生する可能性があることに注意してください。



    入力例３ 3
    2 4 1 6
    1 -2 -2 1
    -2 -5 3 5
    出力例３ 1 0 0 3.86486
    0 1 0 -1.05405
    0 0 1 1.48648

    解が小数になることもあります。
    出力例３では、
    １列目： 143 37 ,２列目： -39 27 ,３列目： 92 37 になっています。


    補足：行基本変形について

    以下３つのみを使って連立方程式を解くこと、
    機械的に解くことが可能な解き方がある。（調べると他にも沢山解き方があった。）

    • １つの行に”０”でない定数をかける。
    • １つの行の定数倍を別の行に加える。
    • ２つの行を入れ替える。
